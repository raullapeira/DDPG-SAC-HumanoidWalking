**Purpose:** Deep RL project teaching a 17-DOF humanoid robot to walk in MuJoCo simulation.

### Structure
```
DDPG-SAC-HumanoidWalking/
├── main_sac.py       # SAC training (primary algorithm)
├── main.py           # DDPG training (baseline)

├── utils/

├── results/          # Training curve plots
└── media/            # humanoid_walking.gif demo
```

### Algorithms
| | **SAC** | **DDPG** |
|---|---|---|
| Policy | Stochastic Gaussian | Deterministic + OU noise |
| Q-networks | Twin Q (reduces overestimation) | Single Q |
| Exploration | Entropy regularization (auto alpha) | OU noise (σ decays 0.3→0.05) |
| Convergence | ~20k steps | ~2x slower |

### Architecture
- **State space:** 376-dim (Gymnasium Humanoid-v5)
- **Action space:** 17 continuous joint torques
- **Networks:** 400→300 hidden layers (ReLU), tanh-bounded output
- **Reward shaping:** survival bonus, forward velocity (2x), standing bonus (+2 if torso height >1m)

### Tech Stack
- **Environment:** `gymnasium` Humanoid-v5 (MuJoCo)
- **Framework:** PyTorch + CUDA
- **Logging:** TensorBoard (SAC), matplotlib live plots (DDPG)

### Performance
- SAC achieves average reward **6000+** post-convergence
- Agents walk stably for **8+ hours** without falls
- Author: Ritwik Rohan (Johns Hopkins MSE '25)

The neural network outputs actions in `[-1, 1]`. Those get mapped to actual joint angles. The question is: **what does action=0 mean physically?**

The old code used:
